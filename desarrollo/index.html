Estructura de Archivos

La aplicación principal consta de diversos archivos de cabecera como List.h, Nodo.h (figura), ReadParameters.h, NFP.h, NFR.h y BL.h, los cuales son incluidos y distribuidos de manera homogénea para la eficiencia de memoria que es el principal objetivo en cumplir. Además, contempla una sucesión de archivos .cc que instancian los objetos de los archivos de cabezera y permiten a la contrucción del archivo main. Por último, se genera el archivo Makefile que permite la unión de los archivos divididos en carpetas y creación de los archivos objetos (no confundir los objetos de código), permitiendo el surgimiento del binario (programa). A continuación se presenta arquitectura de archivos. 

	ISP/
	 |	Makefile
	 |	src/
	 |	 |	Include/
	 |	 |	 |	List.h
	 |	 |	 |	Nodo.h
	 |	 |	 |	ISP.h
	 |	 |	 |	ReadParameters.h
	 |	 |	 |	NFP.h
	 |	 |	 |	NFR.h
	 |	 |	 |	BL.h
	 |	 |	Object/ (Almacenan todos los archivos objetos generados por el compilador)
	 |	 |	*.cc (Diversos archivos C++)
	 |	 |	Makefile
	 |	ISP

Los archivos de cabezera, contienen la importancia del conjunto principal de estructura de datos del sistema y que serán expuestos y explicados brevemente su intencion y funcionamiento en la aplicación.
